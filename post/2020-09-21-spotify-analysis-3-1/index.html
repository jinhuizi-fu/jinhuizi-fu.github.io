Spotify Analysis 3.1 - What Makes a Playlist Successful?

By Jinhuizi Fu

November 2, 2020

I already wrote two posts on Spotify analysis - one on the Beatles tracks and albums, and another one for my own listening habits. I had a lot of fun doing these exploratory data analysis, and I decide to make it a trilogy. In this new post, I‚Äôm going to take one step further: I want to find out what makes a playlist successful, and try to predict how successful a playlist will be based on its known features. (Future idea may be to generate a playlist that might be successful.) Since it will be fairly long, I will split into 2 parts. Part 1 (this article) will be focusing on getting the data and EDA, part 2 will build 2 models and compare the results.

Define ‚Äúsuccessful‚Äù

So we need to define ‚Äúsuccessful‚Äù for a playlist, and the number of followers seems like a good one to use. It‚Äôs very straightforward. (If we define ‚Äúsuccessful‚Äù this way, my most successful playlist on Spotify has 6 followers. Yay! üòÇ)

I didn‚Äôt mention my own ‚Äúsuccess‚Äù just for fun. It was also pointing us to a direction - we need to carefully choose a sample data that has enough information for us to mine the underlying trends. In this analysis, I decided to focus on Spotify featured playlists, i.e.¬†created by Spotify.

Brainstorming

Now that we have our response variable: number of Followers, some questions comes to mind:

  • What‚Äôs the distribution of it?
  • What might be a contributing factor for a playlist to be successful?

For the 1st question, we can look at some basic statistics.

For the 2nd question, some brainstorming ideas as follows:

  • Number of tracks
  • Genre (to quantify, maybe pop%, EDA%, genre count, etc.)

Download track features in each playlist

Based on the previous brainstorming, I decide to download the track names, audio features, and artist information from each playlist. Because genre information is not available for each track but only for each artist, it is too time consuming at the time. So for the time being, we are only checking if the artist is in top 10 or top 20 followed artists.

Also when downloading, we have to run in loops. So we are creating features at the same time when we download these data, follow the list and save them into a data frame.

Some notes here worth mentioning: there are a few playlists that cannot get required track audio feature information, so the whole loop will fail. In this case we add an error handling code to replace these errors with the information of the first playlist on the list, and we can de-duplicate later.

library(data.table)

# define top 10 and top 20 (11~20) most followed artists
top10_artist = c("Ed Sheeran","Ariana Grande","Drake","Rihanna","Eminem","Justin Bieber",
                 "Billie Eilish","Taylor Swift","Imagine Dragons","Shawn Mendes")
top20_artist = c("Queen","Post Malone","Coldplay","Maroon 5","Bruno Mars",
                 "Marshmello","Bad Bunny","Beyonc√©","Ozuna","The Weekend")

pt_top10_artist = paste(top10_artist, collapse="|")
pt_top20_artist = paste(top20_artist, collapse="|")

list1 = spotify_pl$id

# replace error with the first row - we can de-duplicate later!
#test_feature = get_playlist_audio_features(username, spotify_pl$id[1])

# Create features for each 
# compute mean and standard deviation of the track audio features
# error handling - replace with a value if an error occurs instead of hault the whole process
.f = function() {
  spotify_pl_feature <-
    length(list1) %>%
    seq() %>%
    map(function(i) {
      tryCatch(get_playlist_audio_features(username, list1[i]),
               error = function(e){test_feature}) %>%
        select(playlist_id,track.artists,track.album.release_date,track.popularity,danceability,
             valence,loudness,energy,speechiness,acousticness,instrumentalness,liveness,tempo) %>%
        rowwise %>% 
        mutate(artists_name = paste(track.artists$name, collapse = ', '),
             track_age = 2020-as.numeric(substr(track.album.release_date,1,4)),
             top10_ind = grepl(pt_top10_artist,artists_name),
             top20_ind = grepl(pt_top20_artist,artists_name)) %>% 
        group_by(playlist_id) %>% 
        summarise(track_count = n(),
                top10_pct = sum(top10_ind)/n(),
                top20_pct = sum(top20_ind)/n(),
                mean_track_age = mean(track_age),
                mean_popularity = mean(track.popularity),
                sd_popularity = sd(track.popularity),
                mean_danceability = mean(danceability),
                sd_danceability = sd(danceability),
                mean_valence = mean(valence),
                sd_valence = sd(valence),
                mean_loudness = mean(loudness),
                sd_loudness = sd(loudness),
                mean_energy = mean(energy),
                sd_energy = sd(energy),
                mean_speechiness = mean(speechiness),
                sd_speechiness = sd(speechiness),
                mean_acousticness = mean(acousticness),
                sd_acousticness = sd(acousticness),
                mean_instrumentalness = mean(instrumentalness),
                sd_instrumentalness = sd(instrumentalness),
                mean_liveness = mean(liveness),
                sd_liveness = sd(liveness),
                mean_tempo = mean(tempo),
                sd_tempo = sd(tempo)) 
    }) %>% 
  reduce(rbind)
}

# remove duplicate rows

Explore the Dataset

Now that we have our data downloaded and feature created, let‚Äôs do some EDA on the data we‚Äôve got!

Response Variable

Our definition of ‚Äúsuccessful‚Äù is the number of followers in our dataset. We have removed the missing from our dataset, now let‚Äôs check its distribution:

library(ggplot2)
ggplot(spotify_pl_feature_2, aes(x=spotify_pl_feature_2$Followers)) + 
  geom_histogram() +
  #geom_density() + 
  xlab("Followers") + 
  ggtitle("Distribution of Followers")

The distribution of the followers is very skewed to the right, meaning mostly concentrated with a smaller number of followers, but there are a few outliers that have a looooooot of followers.

Check if the log-transformed form is a Normal distribution:

ggplot(spotify_pl_feature_2, aes(x=log(spotify_pl_feature_2$Followers))) + 
  geom_histogram() +
  #geom_density(alpha=.2, fill="#FF6666") + 
  xlab("Followers") + 
  ggtitle("Distribution of Followers")

After log-transformation, although it‚Äôs not perfectly ‚Äúbell-shaped‚Äù, it‚Äôs very much better than before. Being closer to a ‚ÄúNormal distribution‚Äù after the transformation makes it more robust, and will better fit the assumption of a Normally distributed response variable for Linear Regression.

Independent Variables

Some of the variables have NA‚Äôs in it. I decide to impute them with median because 1) the number of missing is relatively small (20 out of 1233) so impact is not very big, and 2) median is less affected by skewness of the distribution than mean.

spotify_pl_feature_3 <- data.frame(lapply(spotify_pl_feature_2,function(x) {
    if(is.numeric(x)) ifelse(is.na(x),median(x,na.rm=T),x) else x}))
